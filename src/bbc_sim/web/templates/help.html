基本概念

bbc-sim は SBCO 標準ポイントリストから 仮想 B-BC（BACnet B-BC デバイス） を生成し、 BACnet/IP で公開します。実設備なしに上流システム（Building OS / Eclipse Hono / Ditto）との 結合試験が可能です。

動作モード

simulator
内部で値を自動生成（Random Walk / Sinusoidal など）。南向きバインディング不要。上流システムの開発・試験に使用。
gateway
南向きプロトコルから受け取った実データを BACnet/IP に反映。実フィールドデータをシミュレートなしに上流へ橋渡し。
combined
オブジェクト単位で simulator と gateway を混在。一部は内部生成、一部は実データバインド。

⚠️ モードの変更にはプロセス再起動が必要です（破壊的操作参照）。

デバイス/オブジェクト画面

管理中の B-BC デバイスとすべての BACnet オブジェクトを一覧表示します。 bbc_id（B-BC 識別子）と device_id（BACnet Device Instance）は 別の値です（ADR-003）。

オブジェクト名をクリックすると詳細画面で値の書込みや Fault 注入ができます。

バインディング画面

gateway / combined モード時、各オブジェクトの南向きバインディング状態を表示します。

  • protocol: mqtt / zeromq / wot / grpc
  • direction: telemetry（南→北）/ command（北→南）/ both
  • 品質: good（正常受信）/ bad（最終受信がエラー）/ unknown（未受信）

接続状態画面

北向き BACnet/IP の稼働状態をローカル内省で表示します。 上流（Hono/Ditto/Building OS）へは直接問い合わせをしません（ADR-005）。

  • Bind 状態: プロセス起動時に BACnet/IP ソケットへバインドできたか
  • BACnet 要求カウンタ: Who-Is / ReadProperty / WriteProperty などの受信数（プロセス起動後の累計）
  • BBMD / Foreign Device: 設定値（simulator.yaml の network セクション）

ログ画面

プロセス内のリングバッファから直近のログ（最大 1000 件）を表示します。 レベルフィルタと自動更新（5 秒）で稼働中の状態を確認できます。

捕捉される主なカテゴリ: 検証エラー / バインディング接続失敗 / telemetry 受信エラー / Fault 活動 / BACnet サービス例外

点リスト画面

実行中の simulator.yaml を再読込して差分をライブ適用します。

  • 検証: 再読込前に YAML を検証し、エラーがある場合は何も変更しません。
  • ライブ適用: 値/メタデータ/writable/update の変更、オブジェクトの追加・削除を適用。
  • 再起動必要: object_type / object_instance / network 設定の変更はプロセス再起動が必要。

注: bbc-sim run --source-path <yaml> を指定しないと再読込できません。

破壊的操作の注意

操作影響範囲回復方法
値の書込み対象オブジェクトの presentValue が変わる。北向き ReadProperty で上流に見える。再度書込みか YAML から再起動で初期値に戻る。
Fault 注入statusFlags / outOfService / eventState が変わる。上流に異常として見える。fault: clear で正常化。
OutOfServiceオブジェクトが Out-of-Service 状態に。書込みは引き続き受け付ける。fault: clear で解除。
点リスト再読込オブジェクトの追加・削除・属性変更。削除されたオブジェクトは即座に消える。YAML を元に戻して再読込か再起動。
モード変更simulator ↔ gateway ↔ combined は再起動が必要。CLI で --mode を指定して再起動。

典型的な運用フロー

  1. SBCO CSV から YAML を生成: bbc-sim generate-yaml --input list.csv --output config.yaml --bbc-id bbc-001 --bacnet-device-id 1001
  2. 起動: bbc-sim run --config config.yaml --rest-port 8080 --ui
  3. Dashboard でバインド状態とオブジェクト数を確認。
  4. デバイス/オブジェクト画面で値と型を確認。
  5. YABE などの BACnet Explorer から北向き接続を検証（接続状態画面でカウンタ確認）。
  6. 異常系試験: オブジェクト詳細画面から Fault を注入し、上流の挙動を確認。
  7. 点リスト更新: YAML を編集し点リスト画面から再読込。
